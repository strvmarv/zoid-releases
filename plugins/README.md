# Contributing a plugin

This directory holds one TOML manifest per plugin, plus a generated
`index.json` that lists them all.

## Adding a plugin

1. Run the Spec 1 converter (`zoid-plugin-import`) against the upstream
   source you want to package, e.g.:

   ```sh
   zoid-plugin-import --repo <owner>/<repo> --ref <commit-sha> --out plugins/<id>.toml
   ```

   This produces a `<id>.toml` manifest in the same `schema = 1`
   `[plugin]` / `[source]` / `[mode]` (or `[skills]`) shape used by the
   other manifests in this directory.

2. Open a pull request adding just your `plugins/<id>.toml` file.

3. Do **not** hand-edit `plugins/index.json`. It is generated by CI
   (`.github/workflows/catalog-index.yml`) from every `*.toml` in this
   directory whenever one changes on `main`, and is committed back
   automatically. Any manual edits to `index.json` will be overwritten
   the next time a manifest changes.

## Manifest requirements

Each `<id>.toml` must have, at minimum:

- `[plugin]` with `id`, `schema = 1`, `kind` (e.g. `["mode"]` or
  `["skills"]`), `name`, and `description`.
- `[source]` with `repo` (the upstream `owner/repo`) and `ref` (a pinned
  commit SHA — not a branch or tag, so the manifest is reproducible).
- A `license` field under `[plugin]` is optional but encouraged when the
  upstream license is known.

See `plugins/superpowers.toml` for a complete example.
